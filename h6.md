## Schneier 2015: Applied Cryptography — 2.3 One-Way Functions
A one-way function is easy to compute in one direction but computationally infeasible to reverse. Example: breaking a plate is easy; reassembling it from shards is not.
No one has mathematically proven one-way functions exist, but many functions behave this way in practice.
They are the foundation of public-key cryptography.
A trapdoor one-way function has a secret shortcut — easy to reverse if you know the secret (the private key). This is the basis of asymmetric encryption (e.g., RSA).

## Schneier 2015: Applied Cryptography — 2.4 One-Way Hash Functions
A one-way hash function (digest/fingerprint) maps variable-length input to a fixed-length output (e.g., 256 bits).
Properties: easy to compute the hash; computationally infeasible to reverse or find two inputs with the same hash (collision resistance).
A small change in input produces a completely different hash (avalanche effect).
Used for: integrity checks, password storage, digital signatures, MACs.
A Message Authentication Code (MAC) is a hash with a secret key — proves both integrity and authenticity.
